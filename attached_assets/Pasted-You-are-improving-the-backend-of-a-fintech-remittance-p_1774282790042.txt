1️⃣ SETTLEMENT SERVICE

Create:

src/services/settlementService.ts

Purpose:
Track how much Habeshare owes or is owed by each partner.

Core concept:
Net settlement instead of per-transaction settlement.

Functions:

1. recordSettlement(tx)

Input:

- txId
- provider (CHAPA / TELEBIRR / BANK)
- amount
- currency
- direction ("inflow" | "outflow")

Logic:

- aggregate partner balances
- store running totals

2. getPartnerBalance(provider)

Return:

{
provider: "CHAPA",
currency: "ETB",
netBalance: 1200000
}

3. generateSettlementSummary(date)

Return:

{
provider: "CHAPA",
totalInflow: 5000000,

}

3️⃣ RECONCILIATION SERVICE

Create:

src/services/reconciliationService.ts

Purpose:
Compare internal transactions vs partner data.

Functions:

1. reconcileTransactions(date)

Steps:

- fetch all transactions (COMPLETED)
- group by provider
- compare with settlement records
- detect mismatches

2. detectMismatch(tx, providerData)

Mismatch conditions:

- amount mismatch
- missing transaction
- duplicate transaction
- status mismatch
