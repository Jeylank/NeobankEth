Add a production-safe "Risk Controls Layer" to the Habeshare backend before beta launch.

Important:
- This is a backend-only safety feature set.
- Do NOT break existing payout, wallet, FX marketplace, treasury, or remittance logic.
- Integrate cleanly into the current Node.js + TypeScript + Express + Firebase Admin + Firestore architecture.
- Use strict TypeScript.
- Keep all existing flows working, but enforce safety controls before risky actions execute.

Build these 4 categories:

1. Global kill switch
2. Transaction / velocity limits
3. Safety guards for risky operations
4. Admin management endpoints

--------------------------------
1) GLOBAL KILL SWITCH
--------------------------------

Create service:

src/services/riskControls/killSwitchService.ts

Purpose:
Allow admins to instantly pause critical operations across the platform.

Supported kill switches:
- remittance_enabled
- wallet_topup_enabled
- recurring_support_enabled
- campaign_payout_enabled
- fx_marketplace_enabled
- referral_rewards_enabled

Store config in Firestore collection:

system_controls

Example document:
{
  key: "remittance_enabled",
  enabled: true,
  updatedAt: timestamp,
  updatedBy: "admin_uid",
  reason: "optional text"
}

Functions:
- isEnabled(controlKey)
- setControl(controlKey, enabled, adminUid, reason)
- getAllControls()

Behavior:
- before executing remittance, wallet top-up, FX quote selection, recurring support, or campaign payout, check the relevant kill switch
- if disabled, return a structured error:
  code: "FEATURE_DISABLED"
  message: "This service is temporarily unavailable"

--------------------------------
3) SAFETY GUARDS FOR RISKY OPERATIONS
--------------------------------

Create service:

src/services/riskControls/safetyGuardsService.ts

Purpose:
Run pre-flight checks before risky financial actions.

Checks to implement:
- user KYC must be verified before remittance above threshold
- wallet must have sufficient available balance
- selected FX quote must still be valid
- treasury reservation must exist before payout execution
- provider must be healthy
- provider liquidity must be sufficient
- user account must not be frozen or blocked
- repeated failed transfers should trigger REVIEW_REQUIRED
- optional biometric confirmation flag can be required for high-value transfers

Create Firestore collection:

risk_flags

Example document:
{
  userId: "uid123",
  isFrozen: false,
  isBlocked: false,
  reviewRequired: false,
  reason: null,
  updatedAt: timestamp
}

Functions:
- validateUserRiskState(userId)
- validatePayoutPreconditions(txId, userId)
- requireEnhancedKycIfNeeded(userId, amount, currency)
- markReviewRequired(userId, reason)
- freezeUser(userId, reason)
- unfreezeUser(userId)

Behavior:
- if review required, transaction should not proceed automatically
- return:
  code: "REVIEW_REQUIRED"
  message: "This transaction requires manual review"
