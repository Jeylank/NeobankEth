==================================================
5) RECONCILIATION LOGIC
==================================================

Implement these rules:

A) MATCHED
If:
- providerRef exists on both sides
- amount matches
- status matches

B) AMOUNT_MISMATCH
If:
- tx exists both sides
- amount differs

C) STATUS_MISMATCH
If:
- tx exists both sides
- internal status differs from provider status

D) MISSING_EXTERNAL
If:
- tx exists internally
- no matching provider entry found

E) MISSING_INTERNAL
If:
- provider entry exists
- no internal tx found

F) DUPLICATE_PAYOUT
If:
- same providerRef or same tx characteristics appear multiple times

G) STALE_RESERVATION
If:
- reservation is still active
- transaction is completed / failed / expired
- reservation was not confirmed or released

H) SETTLEMENT_OVERDUE
If:
- settlement obligation remains open beyond configured threshold

I) LEDGER_INCONSISTENCY
If:
- wallet debit exists but payout never executed
- payout completed but no ledger debit/settlement entry exists
